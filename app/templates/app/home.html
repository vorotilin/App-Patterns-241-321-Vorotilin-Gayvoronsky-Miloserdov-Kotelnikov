Платежей

Порождающие паттерны
  • Singleton — Logger: единственный экземпляр
  • Abstract Factory — заказ + платёж + трекинг одной фабрикой
  • Decorator — динамическое добавление услуг к заказу
Поведенческие паттерны
  • Strategy — расчёт цены по тарифу
  • State — жизненный цикл заказа
  • Template Method — шаблонный процесс доставки
  • Observer — уведомление подписчиков
  • Command — команды с поддержкой Undo
  • Iterator — обход иерархии заказов
Структурные паттерны
  • Proxy — контроль доступа по роли
  • Adapter — SMS / Email через единый интерфейс
  • Facade — единая точка входа в систему
  • Composite — дерево заказов по тарифу/статусу
  • A* (Dijkstra) — кратчайший маршрут доставки
Бизнес-процессы
  1. БП1: Создание и обработка заказа — клиент создаёт заказ → оплата → назначение курьера → доставка
  2. БП2: Маршрутизация — выбор кратчайшего пути доставки через граф узлов (A*)
